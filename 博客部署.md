# Hugo 博客部署指南

## 项目概述
- **位置**: `d:\PCCodes\blog\hugo-site`
- **Hugo 版本**: v0.160.1 extended
- **本地服务器**: http://localhost:1313
- **特性**: 无数据库、无编程、完全 Markdown 驱动

---

## 关键步骤

### 1. 启动本地开发服务器

在 PowerShell 中运行以下命令（需进入项目目录）：

```powershell
cd d:\PCCodes\blog\hugo-site
.\bin\hugo.exe server -D -p 1313
```

**说明**：
- `-D` 选项：显示草稿文章（`draft = true` 的文章也会显示）
- `-p 1313` 选项：指定端口为 1313
- 运行后访问 http://localhost:1313 即可预览站点

### 2. 创建新文章

在 PowerShell 中运行（项目目录下）：

```powershell
.\bin\hugo.exe new posts/文章标题.md
```

**示例**：
```powershell
.\bin\hugo.exe new posts/我的第一篇博客.md
```

**生成的文件位置**: `content/posts/我的第一篇博客.md`

### 3. 编辑文章

用任意文本编辑器（VS Code、Notepad++ 等）打开 `content/posts/` 下的 `.md` 文件。

**文章格式示例**：
```markdown
+++
title = "我的第一篇文章"
date = 2026-04-18T12:00:00+08:00
draft = false
tags = ["标签1", "标签2"]
description = "这是文章描述"
+++

## 文章标题

这是文章正文，使用 Markdown 格式编写。

### 二级标题

支持：
- **粗体**
- *斜体*
- `代码`
- [链接](https://example.com)
- ![图片](/images/demo.jpg)

```python
# 代码块
def hello():
    print("Hello, Hugo!")
```
```

**关键字段说明**：
| 字段 | 说明 |
|------|------|
| `title` | 文章标题（显示在浏览器和列表中） |
| `date` | 发布日期（格式: `YYYY-MM-DDTHH:MM:SS+08:00`） |
| `draft` | `false` 发布，`true` 草稿（仅 `-D` 时显示） |
| `tags` | 标签数组，用于分类 |
| `description` | 文章描述（列表页显示） |

### 4. 文章自动刷新

保存文章文件后，Hugo 会自动：
- 重新生成 HTML
- 刷新浏览器页面

无需重启服务器！

### 5. 生成静态网站

部署到服务器时，构建完整站点：

```powershell
# 生成包含草稿的站点（仅用于本地测试）
.\bin\hugo.exe -D

# 生成正式站点（不含草稿）
.\bin\hugo.exe
```

生成的文件在 `public/` 目录，可上传到任何静态网站托管服务（GitHub Pages、Netlify 等）。

## 修改网站配置

编辑 `config.toml` 修改以下内容：

```toml
baseURL = "http://localhost:1313/"    # 网站 URL（部署时改为实际域名）
languageCode = "zh-CN"                # 语言代码
title = "我的 Hugo 博客"              # 网站标题
paginate = 10                         # 每页文章数

[menu]
[[menu.main]]
name = "首页"
url = "/"
weight = 1

[[menu.main]]
name = "文章"
url = "/posts/"
weight = 2
```

修改后保存，重启服务器生效。

## 常见问题

### Q: 启动服务器提示"端口已被占用"
**A**: 修改 `-p` 参数为其他端口
```powershell
.\bin\hugo.exe server -D -p 1314
```

### Q: 修改配置后页面未更新
**A**: 重启服务器（关闭现有进程，重新运行启动命令）

### Q: 发布的文章不显示
**A**: 检查 `draft` 字段是否为 `false`，或启动时加 `-D` 选项

### Q: 如何添加图片
**A**: 
1. 将图片放在 `static/images/` 目录
2. 在 Markdown 中引用：`![描述](/images/文件名.jpg)`

### Q: 构建后如何部署
**A**: 
1. 运行 `.\bin\hugo.exe` 生成 `public/` 目录
2. 将 `public/` 目录内容上传到服务器或 GitHub Pages

## 部署到 GitHub Pages（公网访问）

### 前置要求
- GitHub 账户（免费注册）
- Git 已安装（[下载](https://git-scm.com/)）
- Hugo 项目（本地已完成）

### 第 1 步：创建 GitHub 仓库

**方式 A：创建用户Pages仓库（推荐）**
1. 登录 [GitHub](https://github.com)
2. 点击右上角 `+` → `New repository`
3. 仓库名设为：`username.github.io`（将 `username` 替换为你的GitHub用户名）
4. 选择 `Public` 并勾选 "Add a README file"
5. 点击 `Create repository`

**方式 B：创建项目Pages仓库**
1. 创建任意仓库名（如 `my-blog`）
2. 部署后访问 `https://username.github.io/my-blog`

### 第 2 步：更新本地配置

修改 `config.toml` 中的 `baseURL`：

**方式 A 用户Pages：**
```toml
baseURL = "https://username.github.io/"
```

**方式 B 项目Pages：**
```toml
baseURL = "https://username.github.io/my-blog/"
```

### 第 3 步：构建静态文件

在项目根目录运行：
```powershell
cd d:\PCCodes\blog\hugo-site
.\bin\hugo.exe
```

这会在 `public/` 目录生成完整网站。

### 第 4 步：初始化本地 Git 仓库

```powershell
# 初始化 Git
git init

# 添加所有文件到暂存区
git add .

# 创建初始提交
git commit -m "Initial Hugo blog setup"
```

### 第 5 步：连接远程仓库并推送

```powershell
# 添加远程仓库（替换 username）
git remote add origin https://github.com/username/username.github.io.git

# 推送到远程仓库
git branch -M main
git push -u origin main
```

### 第 6 步：启用 GitHub Pages

1. 在 GitHub 仓库页面进入 `Settings` → `Pages`
2. **Source** 选择 `Deploy from a branch`
3. **Branch** 选择 `main` 和 `/root` 目录
4. 点击 `Save`
5. 稍等几分钟后，页面顶部会显示：`Your site is live at https://username.github.io/`

### 第 7 步：验证部署

访问 `https://username.github.io/` 查看你的博客！

## 配置自定义域名（可选）

如果你有自己的域名，可以将其指向 GitHub Pages：

### 方式 1：CNAME 配置（推荐）

1. 编辑域名 DNS 设置，添加 CNAME 记录：
   ```
   主机名: www
   指向: username.github.io
   ```

2. 在项目 `static/` 目录创建 `CNAME` 文件（无扩展名），内容为：
   ```
   yourdomain.com
   ```

3. 提交并推送：
   ```powershell
   git add .
   git commit -m "Add custom domain"
   git push
   ```

4. GitHub Pages 设置中，在 `Custom domain` 输入你的域名

### 方式 2：A 记录配置

1. 编辑域名 DNS，添加 A 记录指向 GitHub IP：
   ```
   185.199.108.153
   185.199.109.153
   185.199.110.153
   185.199.111.153
   ```

2. 同样在 GitHub Pages 设置中输入域名

稍等 5-30 分钟 DNS 生效，即可通过自定义域名访问。

## 常见问题

### Q: 推送后页面还是旧的
**A**: 
- 清除浏览器缓存（Ctrl+Shift+Delete）
- 等待 2-5 分钟让 GitHub Actions 完成部署
- 查看仓库 `Actions` 标签页确认部署状态

### Q: 404 页面找不到
**A**: 
- 确保 `config.toml` 中的 `baseURL` 设置正确
- 重新运行 `.\bin\hugo.exe` 构建
- 确认 `public/` 目录已推送到 GitHub

### Q: 如何只部署 public 目录
**A**: 创建 `.gitignore` 文件，只跟踪 `public/` 目录：
```
/*
!/public/
!.gitignore
```

或者，使用 GitHub Actions 自动化部署（见下一章节）

## 自动化部署（GitHub Actions）- 高级选项

不想手动每次构建，让 GitHub 自动生成？使用 GitHub Actions：

### 第 1 步：创建工作流文件

在项目根目录创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy Hugo

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: '0.160.1'
          extended: true
      
      - name: Build
        run: hugo
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
```

### 第 2 步：提交并推送

```powershell
git add .github/workflows/deploy.yml
git commit -m "Add GitHub Actions deployment"
git push
```

### 第 3 步：配置 GitHub Pages

1. 仓库 Settings → Pages
2. Source 改为 `Deploy from a branch`
3. Branch 选择 `gh-pages` 分支（Actions 自动创建）
4. 目录选择 `/` (root)

从现在起，每次推送 `main` 分支，GitHub 会自动：
- 构建 Hugo 站点
- 生成 `gh-pages` 分支
- 自动部署到 GitHub Pages

**只需编写 Markdown，不需手动构建！**
